# OWダーツの旅 詳細設計

本書は `docs/basic_design.md` を実装に移すため、詳細設計で決めるべき事項を具体化する。

主な対象は以下。

* スロット演出の具体方式（停止挙動、強調エフェクトの考え方）
* マップルール表示ラベルとリール表示形式
* マップ一覧件数が多い場合のスクロール挙動

なお、抽選ロジックの基本方針や状態管理（state 構造）は基本設計に従う。

---

## 1. 前提・用語

* `mapId`: マップの一意識別子（string）
* `mapRule`: マップルール（string。`maps.yaml` のトップレベルキーをそのまま使う）
* `lastSelectedAt`: そのマップが最後に当選した時刻（ISO 8601文字列、未選択は `null`）
* `mapMaster`: `src/data/maps.yaml` から読み込むマップ定義
  * `maps.yaml` は `mapRule` ごとのオブジェクトを持ち、各トップレベルキー配下の `maps` 配列に `mapId` / `mapName` を定義する
* ルール選択UI: `mapMaster` に存在する `mapRule` のみ表示する
* 「24時間除外」: `now - lastSelectedAt < 24h` の間は抽選対象外

## 2. スロット演出設計

### 2.1 基本方針

* スロット演出の標準時間は、基本設計に定めた標準値を採用する
* 中央当選ラインは、単一リールの中央行とする
* リール移動量の座標系は 1 行ぶんの高さを基準に統一し、停止位置計算と描画位置計算で同じ単位を用いる

### 2.2 速度変化

演出は Web Animations または CSS どちらでもよいが、以下の流れを満たすことを詳細設計として定める。

* フェーズA: 立ち上がり加速
  * 早めに最高速へ乗せるため、短い加速区間とする
* フェーズB: 最高速の維持
  * 候補を目視で追いにくい速度域を一定時間維持する
* フェーズC: 最終停止
  * 当選ラインへそのまま減速し、オーバーシュートせず停止する

イージングの具体値や各フェーズの時間配分は、基本設計の意図を満たす範囲で実装者が調整してよい。

### 2.2.1 高速回転量

最高速区間の体感速度を十分に上げるため、停止位置へ到達する前に追加の周回を入れる。

* 目的: 最高速時に個々の候補が目視で追えない程度のスクロール量を確保する
* 実現方法: 停止位置計算に複数周ぶんの行数オフセットを加算する
* 具体的な周回数は、視認性とテンポのバランスを見て実装者が決定してよい

### 2.3 停止挙動（最適値）

停止時は当選ラインへそのまま着地し、通り過ぎや戻りは行わない。

* 停止方式: 単一アニメーションで `targetIndex` の停止位置へ到達する
* 停止条件: 最終フレームで当選ラインに「ぴったり」揃うこと
* 目的: 結果の即時認識性を高め、停止モーションを簡潔にする

### 2.4 停止時の文字強調エフェクト（最適値）

停止した瞬間に当該アイテムへ、短く軽い強調を入れる。

* 強調はリール停止の直後に短時間だけ行う
* 文字サイズは通常表示より大きめに取り、停止位置の結果が一目で読めること
* エフェクトは、移動・拡大・影・発光のいずれか、または組み合わせでよい
* 強調は読みやすさ優先のため長時間にしない
* 具体的な transform 値、影、発光、イージングは実装者が調整してよい

## 3. リセット時の `ruleFilter` 扱い（最適値）

### 3.1 結論

* リセット時は `mapState`（24時間除外）をクリアする
* `ruleFilter` は表示中の全ルールを ON に戻す（初期状態へ戻すことを詳細設計として確定）

### 3.2 採用理由

* 「リセット = 抽選条件をまっさらな状態へ戻す」という意味を明確にできる
* リセット直後に選択可能なルールが残っているのに抽選できない状態を避けられる

---

## 4. マップルール表示名と表示ラベル

### 4.1 表示方式

* ルール選択UIのチェックボックスラベルは、`maps.yaml` に記載された `mapRule` を大文字化して表示する
* リールおよび結果表示のフォーマット: ``${RuleLabel} : ${mapName.toUpperCase()}``
* リールおよび結果表示では、`RuleLabel` と `mapName` の両方を大文字化する

---

## 5. クールタイム中のマップ一覧件数が多い場合のスクロール挙動（最適値）

### 5.1 スクロール方式

* 左側パネル全体はビューポート高を超えて伸ばさない
* クールタイム一覧パネルはヘッダー・注釈・操作を固定し、マップ名一覧部分のみを縦スクロール可能にする
* 一覧1件あたりの表示は行高が大きく揺れないようにし、最新順の視認性を優先する

### 5.2 描画方針

* 抽選中は一覧の並び替えを即時反映してよいが、UIのカクつきが出る場合は「演出完了後に一括反映」へ切り替えられるよう、状態更新を描画層から分離する

> MVP ではまず単純反映で開始し、性能上の問題が出たら最適化する。

## 6. 抽選ロジック詳細（基本設計準拠の確定事項）

### 6.1 候補の生成順

基本設計に従い、次の順で絞り込む。

1. マップマスタ全件を起点
2. チェック済みルール適用
3. 24時間除外適用
4. 残件から一様乱数で 1 件

### 6.2 リール表示用データ

リール表示は抽選候補配列をそのまま使わず、マップマスタ全件を `RULE : MAP NAME` 形式へ整形した一覧を使う。

* 目的: 抽選前の見た目を固定しつつ、マップルール単位の塊に見えにくいリール順を作る
* 並び順: 画面初回読込時にマップマスタ全件を一度だけシャッフルし、その順をリール順として保持する
* 当選位置計算: `slotResult.mapId` を固定化されたシャッフル済みリール順で探索して `targetIndex` を求める
* 抽選結果決定: 7.1 の候補生成結果に対してのみ乱数選択を行う

## 7. localStorage スキーマ詳細（実装容易性のための確定事項）

### 7.1 保存キー

基本設計の例を踏襲し、次を採用する。

* `ow-darts-journey.mapState`（各 mapId の lastSelectedAt）
* `ow-darts-journey.ruleFilter`（RuleFilter。選択中ルール配列）

### 7.2 値の例（イメージ）

* `mapState`
  * `Record<string, string | null>`
* `ruleFilter`
  * `MapRule[]`

## 8. 状態遷移（UI制御の確定事項）

* マップルール見出しの親チェックボックス:
  * 24時間除外後も候補が残っているルールのみを親チェックボックスの対象とする
  * 対象ルールがすべて `ruleFilter` に含まれるときのみ checked
  * checked 時は disabled とし、操作しても状態変化しない
  * unchecked 時に操作された場合は「候補が残っているルール」をすべて ON に戻す
* 個別ルールチェックボックス:
  * 同一ルール内で抽選可能なマップが 0 件になった場合は自動で OFF にする
  * 上記ルールは disabled とし、再び候補が生じるまで操作不可とする
* 抽選開始ボタン:
  * スロット直下に配置する大型の丸型ボタンとする
  * `slotStatus = spinning` または `slotStatus = stopping` の間は disabled とする
* `CTリセット` ボタン:
  * 一覧ヘッダーと同じ行の右側に配置する
  * クールタイム中のマップが 1 件以上ある場合のみ表示する
  * `slotStatus = spinning` または `slotStatus = stopping` の間は disabled とする
  * 押下して確認ダイアログで確定した後は、一覧中のマップ名と区切り記号だけを短時間フェードアウトさせる
  * その間、一覧パネルの実高さを保持し、フェードアウト完了後の内容高さへ短時間で補間する
  * フェードアウト完了後に `mapState` を空へ更新し、`ruleFilter` を全 ON に戻す
* 注釈ボックス:
  * クールタイム中のマップが 1 件以上ある場合のみ表示する
* 抽選開始時:
  * `slotStatus = spinning`
  * マップルール選択 UI と `CTリセット` ボタンと抽選開始ボタンを非活性
* リール停止時:
  * `slotStatus = stopping`
  * 当選項目の強調演出が終わるまで操作系を非活性のまま維持する
* 強調演出完了時:
  * `slotStatus = result`
  * `slotResult` に対応する項目を停止したリール位置で結果として示す
  * `ruleFilter` から当選した `mapRule` を外す（残り1件だった場合は当選ルールのみ OFF にし、マッププール内に存在する他ルールをすべて ON にする）
  * リール表示は当該抽選回の停止状態を維持し、結果確定直後に消さない
  * 24時間除外の根拠（`mapState`）を更新

---

## 9. 補足（設計の拡張ポイント）

* ルール種類が増えても詳細設計のラベル対応表に追加していけばよい（未知ルールは内部値そのまま表示）
* スロット演出は速度カーブや回転量を定数化しているため、好みの演出に差し替えやすい
